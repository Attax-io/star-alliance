# Star Alliance — Agents

> A guild of **Claude** agents. The Butler is the voice; the Strategist is the
> router; the specialists do the craft. Every member is a Claude model — there is
> no external "doer" layer.

**The canonical instructions live in [`CLAUDE.md`](./CLAUDE.md).** This file is a
short pointer so any agent runner that loads `AGENTS.md` gets the same doctrine.

## The one-paragraph version

Star Alliance is a Claude-only harness plus an MCP server (`mcp/server.py`) that
other projects can use. Members are Claude subagents spawned by the live session
(`subagent_type="the-<member>"`): the Architect runs as **opus**, the Quartermaster
as **haiku**, every other member as **sonnet**, and the Butler is the live session
itself. Bulk or parallel work = fan out several Claude subagents; there is nothing
non-Claude to hand work to. Sources of truth are `star-alliance-members/*.md`,
`star-alliance-skills/<id>/SKILL.md`, `star-alliance-arsenal/models.json`,
`workflows.json`, and `data/domains.json`; everything else is generated by `build.py`
and `guild/install_agents.py`. One hard safety gate remains (`destructive-gate.py`);
everything else is a non-blocking reminder or logger. Read `CLAUDE.md` for the full
doctrine.
