# The fixed self-interrogation

Mined from *Introduction to Self-Knowledge through the Enneagram*. Self-knowledge is built by
answering the **same** honest, gut-level questions every cycle — not by analyzing single events.
Repetition surfaces the patterns that repeat. Answer as the guild, citing log evidence.

Ask the SAME questions every cycle. Record the gut-truth, not the sanitized version.

1. **Recurring failure** — what kind of failure showed up more than once this cycle? (not a single
   incident — the repeating shape)
2. **Trigger** — what condition reliably preceded it? (ambiguous spec, missing context, tool
   timeout, a particular member, a particular skill)
3. **Bias active** — what default did the guild reach for that it should have questioned?
   (over-reading, premature shipping, skipping verification, blind full-reads)
4. **Avoidance** — what did the guild avoid doing that it should have done? (the reflection it
   skipped, the verification it waved through, the escalation it dodged)
5. **Strength used** — which member/skill/doctrine actually carried the wins this cycle?
6. **Gift unused** — what capability sat idle that could have helped? (an unused weapon, a skill
   never loaded, a member never routed to)

> Repeating the same question 20–30 times across cycles forces past the cached self-description
> (*Self-Discovery, Erasmus+*). The first answer is the flattering one; the durable truth shows
> up only on repetition.

Each cycle's answers append to `guild/journal/self-inquiry.md`. Monthly, distil the recurring
answers into proposed edits to member skills or CLAUDE.md doctrine.
