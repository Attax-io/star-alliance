# Heat-Map Analyst

The heat-map-analyst skill audits doc usage across `lex_council/docs/` and proposes Vault Core promotions and archivals based on recent `claude_hits` deltas.

## Inputs

- Read `lex_council/docs/outputs/DOC-HITS-REPORT.md` for current `claude_hits` totals (generated by `bash .claude/scripts/doc-hits-report.sh`).
- For each `.md` file under `lex_council/docs/` (excluding `archived/`, `memory/archived/`, `vault-logs/`, `logs/`, `outputs/`), compute the hit delta over the last 30 days.

## Delta Source

Recent `lex_council/docs/vault-logs/YYYY-MM-DD_housekeeping-run-*.md` logs record per-doc hit counts in their frontmatter or body tables. Cross-reference the last ~30 run logs against the current total to compute deltas. If no delta is available for a doc, treat it as zero.

## Buckets

- **Top 10 hottest** — highest 30-day delta. Propose for promotion in Vault Core (move up the navigation, or add to Task-Oriented Quick Lookup).
- **Bottom 20 coldest with zero changes in 30 days** — propose for `archived/`.
- **Middle tier** — no action.

## Output

Return a structured report with three sections, wikilinks only (no prose descriptions). Keep under 500 words. Do not edit any files — return-only.
